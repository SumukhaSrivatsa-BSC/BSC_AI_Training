# AI Model Health Monitoring System

## Overview
This project analyzes the health and performance of 50 AI/ML models using **NumPy, Pandas, Matplotlib, and Scikit-learn**. It evaluates model accuracy, latency, data drift, CPU usage, prediction confidence, and failure risk.

## Dataset Information
The dataset contains **50 AI models** with the following features:

- **ModelID** – Unique model identifier
- **Algorithm** – Model type (RandomForest, XGBoost, CNN, LSTM, SVM, DecisionTree, Transformer)
- **TrainingSamples** – Number of training samples
- **Accuracy** – Model accuracy (%)
- **InferenceLatency** – Prediction latency (ms)
- **DataDrift** – Data drift percentage
- **CPUUsage** – CPU utilization (%)
- **PredictionConfidence** – Confidence score (0–1)
- **FailureRisk** – Target variable (0 = Low Risk, 1 = High Risk)

## Tasks Performed
### Part A – NumPy
- Created NumPy arrays from dataset columns.
- Calculated mean, median, variance, and standard deviation.
- Found maximum and minimum latency.
- Identified models with high CPU usage and high data drift.
- Normalized training sample values.
- Calculated a custom **Health Score**.

### Part B – Pandas
- Explored dataset structure and statistics.
- Checked missing values and duplicates.
- Analyzed algorithm distribution.
- Grouped and compared performance metrics.
- Classified models as **Healthy, Moderate, or Critical**.
- Exported processed data to CSV.

### Part C – Data Analysis
- Generated correlation matrix.
- Identified factors related to failure risk.
- Compared latency, CPU usage, confidence, and drift.
- Listed healthiest and highest-risk models.

### Part D – Data Visualization
- Accuracy histogram
- Inference latency histogram
- Accuracy vs Data Drift scatter plot
- CPU Usage vs Latency scatter plot
- Algorithm distribution bar chart
- Failure Risk pie chart

### Part E – Machine Learning
- Encoded algorithm names using LabelEncoder.
- Split dataset into training and testing data.
- Trained a Random Forest Classifier.
- Evaluated performance using:
  - Accuracy Score
  - Confusion Matrix
  - Classification Report
- Predicted failure risk for new AI models.

## Key Findings
- Transformer models achieve very high accuracy and confidence.
- Higher data drift is associated with increased failure risk.
- Health Score provides a simple way to evaluate model condition.
- Different algorithms show different CPU and latency characteristics.
- Machine learning can effectively predict failure risk for new models.
